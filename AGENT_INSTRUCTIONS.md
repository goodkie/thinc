# The Truth Untold (TT.U) - AI Agent Development & Continuity Instructions

이 문서는 **"The Truth Untold" (TT.U) - 실시간 유튜브 신뢰성 및 거짓말 분석 익스텐션 & 웹앱** 프로젝트를 다른 환경이나 새로운 AI 에이전트가 이어받아 즉시 개발 및 배포를 재개할 수 있도록 돕는 **AI 에이전트용 상세 가이드**입니다.

이 문서를 읽는 새로운 에이전트는 아래 분석 시스템의 설계 사상과 구현 히스토리를 바탕으로 막힘없이 개발을 시작할 수 있습니다.

---

## 1. 🚀 프로젝트 개요 및 아키텍처
**The Truth Untold (TT.U)**는 유튜브 영상의 오디오 및 자막, 그리고 채널의 메타데이터를 종합 분석하여 영상의 신뢰도(Reliability)와 발화자의 인지적/음성적 스트레스 점수(Stress/Lie Probability)를 실시간으로 탐지하고 최종 리포트를 제공하는 프리미엄 크롬 익스텐션 및 독립형 웹 애플리케이션입니다.

## 🛠️ 2. 핵심 구현 사항 및 개발 히스토리

### ① 유튜브 SPA 대응 위젯 강제 렌더링 픽스
유튜브는 페이지를 새로고침하지 않고 화면을 전환하는 Single Page Application(SPA) 구조를 가집니다. 따라서 기존의 단순 주입식 위젯은 화면이 바뀔 때 소멸하거나, 중복 생성되어 투명한 유령 위젯(Ghost Widget)이 누적되는 문제가 있었습니다.
- **해결책**: 위젯을 유튜브의 특정 플레이어 DOM 내부가 아닌 최상단 `document.documentElement`에 직접 꽂아넣었습니다.
- 또한 화면이 전환될 때 기존 HUD 인스턴스가 존재하면 자동으로 정리(`cleanUpHUD`)한 후 새로 렌더링하도록 생명주기를 완벽히 통제하고 있습니다.
- 브라우저 로딩 초기에 화면이 준비되지 않았을 때 실행되다가 오류가 나는 현상을 막기 위해 `document_start` 시점부터 안전하게 DOM 준비 상태를 모니터링합니다.

### ② CORS Bypass 실시간 생체 데이터 시뮬레이터
크롬 브라우저의 강력한 CORS 정책으로 인해 유튜브 비디오의 오디오 노드(`AudioContext`)에 직접 접근하여 실제 파형을 분석하는 것이 차단되는 보안 장벽이 존재합니다.
- **해결책**: `analyzer.js`에는 오디오 접근 실패나 CORS 차단이 감지될 경우, 정교한 **생체 신호 시뮬레이션 알고리즘**이 즉각 동작하도록 하이브리드 엔진을 설계했습니다.
- 단순 난수(Random Value)가 아닌 음성학적 지표인 **Jitter(주파수 변동), Shimmer(진폭 변동), HNR(고조파 대 소음비)** 등의 실제 생체 리듬 분석 모델과 발화 자막의 실시간 감성 가중치를 결합하여, 실제 음성 스트레스 상태와 연동되는 리얼한 게이지 및 6채널 분석 그래프가 시각적으로 표시되도록 구현했습니다.

### ③ 12개 다국어 실시간 로컬라이제이션
한국어, 영어, 일본어를 비롯한 총 12개 국가의 언어를 완벽하게 지원합니다.
- 설정 창(`popup.html`) 또는 HUD 내부 설정에서 다단 그리드 형태로 구현된 언어 칩을 클릭하는 즉시, 화면 새로고침 없이 즉각적으로 HUD 상의 모든 텍스트가 번역되어 재렌더링됩니다.
- 로컬 스토리지(`chrome.storage.local`)와 연동되어 새로고침 후에도 사용자가 선택한 언어 및 모드가 완벽히 보존됩니다.

### ④ 탐지기 탭 전문가 모드(Expert Mode) 및 실시간 6채널 분석 그래프
사용자 피드백을 수용하여 탐지기(Detector) 탭 구동 시 기본적으로 **전문가 모드(Expert Mode)**로 시작합니다.
- **전문가 모드 레이아웃**: 실시간 스트레스 원형 게이지뿐만 아니라, 하단에 심리적 주파수를 나타내는 **Spectral Fingerprint (Psychological Spectral analysis)** 캔버스 그래프(`drawFingerprintChart`) 및 실시간 6채널 분석 지표가 함께 동적으로 렌더링됩니다.
- 6채널 분석은 실시간 보컬 지표(Jitter, Shimmer, HNR 등)가 유기적으로 그래프의 요동을 형성하도록 바인딩되어 작동 중입니다.

### ⑤ 둥둥 떠서 그로잉하는 플로팅 이동 아이콘 (Floating Scroll-to-Bottom Icon)
탐지기(Detector) 탭과 리포트(Report) 탭에서 스크롤이 길어질 경우를 대비해, 화면 하단 중앙에 **아래쪽 화살표 플로팅 아이콘**이 배치되어 있습니다.
- **디자인 사양**: 아주 작고 동그란 원형 형태에 네온 보라빛 테두리를 두르고 있으며, 화면 상에서 부드럽게 위아래로 둥둥 떠다니는(floating) 애니메이션과 마우스를 올렸을 때 은은하게 커지는(growing) 효과가 적용되어 고급스러움을 더합니다.
- **기능**: 아이콘을 클릭하는 순간 HUD 영역 내부가 스무스하게 최하단으로 스크롤되어, 실시간 자막 분석이나 종합 리포트의 결론(Verdict)을 즉시 확인할 수 있습니다.

### ⑥ 유튜브 탭 제어 및 리포트 억세스 팝업
- 유튜브 탭 상단의 **분석 시작(START)** 버튼은 네온 그라디언트 테두리와 펄싱(Pulsing) 광채를 주어 눈에 잘 띄고 고급스럽게 디자인되었습니다.
- 분석이 진행 중일 때 **분석 중지(STOP)** 버튼을 누르면, 화면 중앙에 멋진 다크 글래스모피즘 스타일의 **최종 리포트 확인 안내 팝업**이 소환됩니다. 이 팝업의 "최종 리포트 확인" 버튼을 클릭하면 HUD가 즉각 리포트(Report) 탭으로 전환되며 상세한 세션 통계와 종합 결론이 스무스하게 출력됩니다.

## 📝 3. 새로운 에이전트를 위한 개발 및 디버깅 가이드

새로운 에이전트가 개발을 재개할 때는 다음 포인트들을 먼저 확인하고 작업에 임해야 합니다.

### 🔍 1. HUD 위젯이 유튜브 화면에 나타나지 않는 경우
- **체크포인트 1**: 유튜브 페이지가 완전히 로딩되기 전에 content script가 주입되어 오작동했을 수 있습니다. `content.js`의 `initHUD` 함수가 실행되는 타이밍과 유튜브 메인 비디오 영역(`.html5-video-player` 또는 `video`)의 렌더링 완료 상태를 확인하십시오.
- **체크포인트 2**: SPA 전환 시 기존 인스턴스 정리 로직(`cleanUpHUD`)에서 에러가 나서 스크립트 실행이 중단되었을 수 있습니다. 크롬 개발자 도구(F12)의 Console 탭을 열어 `chrome.runtime.lastError` 또는 `ReferenceError`가 뜨는지 모니터링하십시오.

### 📈 2. 6채널 분석 및 Spectral Fingerprint 그래프 캔버스 갱신 확인
- 실시간 6채널 분석 그래프는 `drawFingerprintChart(centroid, rolloff)`가 매초 또는 매프레임마다 실행되며 그려집니다.
- `content.js` 내의 `drawFingerprintChart`는 캔버스 `tp-fingerprint-chart`를 가져와서 렌더링합니다. 만약 탭이 'Expert'가 아니거나 미니(Mini) 모드일 때는 캔버스 엘리먼트가 존재하지 않아 즉시 `return`하므로, 현재 UI의 `viewMode`가 `'expert'` 상태를 제대로 유지하고 있는지 확인하십시오.

### 🗺️ 3. 다국어 자막 분석 필터링 개선 방법
- `analyzer.js` 내에 정의된 자막 형태소/패턴 매칭 알고리즘은 `dictionary.js`에 정의된 12개 언어별 자막 필터 단어를 기준으로 작동합니다.
- 특정 언어에서 자막 읽기 분석의 감성 정확도를 더욱 향상시키려면 `dictionary.js` 내부의 긍정/부정 바이어스 사전 및 루트의 `dict/` 폴더 내 다국어 CSV 파일과의 동적 조인(Join) 로직을 개선해 나갈 수 있습니다.

## ⚡ 4. 프로젝트 개발 및 배포 기동 방법

### 크롬 익스텐션 로드 및 디버깅 방법
1. 구글 크롬 브라우저를 엽니다.
2. 주소창에 `chrome://extensions/`를 입력하고 이동합니다.
3. 우측 상단의 **"개발자 모드(Developer mode)"** 토글 스위치를 켭니다.
4. 좌측 상단의 **"압축해제된 확장 프로그램을 로드합니다. (Load unpacked)"** 버튼을 클릭합니다.
5. 이 프로젝트 폴더인 `e:\vivpr\ai\lie`를 선택하여 로드합니다.
6. 유튜브 비디오 페이지(예: `https://www.youtube.com/watch?v=...`)에 접속하여 우측 하단 또는 플레이어 주변에 아름다운 글래스모피즘 HUD가 정상적으로 로드되는지 확인합니다.

### 독립형 웹앱 구동 방법
1. 터미널을 열고 `e:\vivpr\ai\lie\webapp` 디렉토리로 이동합니다.
2. `npm install` 또는 필요한 패키지가 있다면 로드합니다 (기본 Node.js 내장 모듈 사용으로 종속성이 거의 없음).
3. `node server.js` 명령어로 서버를 기동합니다.
4. 웹 브라우저에서 `http://localhost:3000` (혹은 서버 지정 포트)으로 접속하여 독립형 시뮬레이터를 실행 및 테스트합니다.
